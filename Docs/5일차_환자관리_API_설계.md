## [API-PTNT-001] 환자 정보 등록

### 1. API 개요

| 항목               | 내용                                                              |
| ------------------ | ----------------------------------------------------------------- |
| **API 이름**       | 환자 정보 등록 API                                                |
| **설명**           | 필수 인적사항을 입력받아 시스템에 환자를 신규 등록 (REQ-PTNT-001) |
| **엔드포인트**     | `/apis/patients`                                                  |
| **메서드**         | `POST`                                                            |
| **인증 필요 여부** | Y (사내 의료인 권한)                                              |

### 2. 요청 (Request)

- **Headers**
  | Key | Value | 설명 |
  | --- | --- | --- |
  | `Content-Type` | `application/json` | 요청 데이터 타입 |
  | `Authorization` | `Bearer <access_token>` | 사내 의료인 JWT 인증 토큰 |

- **본문 예시 (Body)**

```json
{
	"name": "이정재",
	"age": 45,
	"gender": "MALE",
	"phone": "010-1234-5678"
}
```

- **본문 필드**
  | 파라미터명 | 타입 | 필수(Y/N) | 설명 |
  | --- | --- | --- | --- |
  | `name` | string | Y | 환자 이름 (최소 2자, 최대 30자) |
  | `age` | int | Y | 환자 나이 (0 ~ 150세 범위 제약) |
  | `gender` | string | Y | 성별 (`MALE` 또는 `FEMALE` 고정형 열거형) |
  | `phone` | string | Y | 휴대폰 번호 (하이픈 포함 가능, 내부 정제됨) |

### 3. 응답 (Response)

- **성공 (201 Created)**

```json
{
	"name": "이정재",
	"gender": "MALE",
	"phone": "01012345678",
	"age": 45
}
```

- **실패 (422 Unprocessable Entity)**

```json
{
	"detail": "입력값 검증 실패: 올바른 성별 형식을 지정하거나 나이 범위를 준수하십시오."
}
```

## [API-PTNT-002] 환자 목록 조회 및 검색

| **엔드포인트**     | `/apis/patients`                                                    |
| **메서드**         | `GET`                                                               |

    "gender": "MALE",
    "phone": "01012345678",
    "created_at": "2026-06-10T21:40:00Z",

  }
]
| 필드명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `id` | long | 데이터베이스 환자 고유 식별 고유 ID (bigint) |

### 4. 비고 (Notes - 공통 규칙)

1. **전화번호 정제 메커니즘:** 클라이언트가 요청하는 모든 `phone` 값 내부의 하이픈(`-`)과 공백 문자열은 백엔드의 Pydantic 검증 로직 통과 시 자동으로 일괄 제거 가공됩니다. 디바이스 데이터베이스 유효 제약조건이 `varchar(11)`이므로, 응답 레이어 및 조회 데이터 레이어에는 숫자 평문 11자리(예: `01012345678`)만 반환됨을 전제하고 UI를 설계해야 합니다.
2. **성별 규격 제한:** `gender` 필드는 데이터베이스 Native Enum 타입과 매핑됩니다. 명세된 문자열 대문자 `MALE`과 `FEMALE` 이외의 소문자 혹은 다른 문자열 유입 시 즉각 시스템 진입부에서 차단됩니다.
3. **비동기 파일 삭제 보증:** 환자 삭제 처리 시 연동된 파일 청소 메커니즘은 성능 요구사항(3초 이내 응답)을 위해 즉시 동기화 처리를 수행하지 않고, 서버 백그라운드 스케줄 작업으로 격리 수행됩니다.
